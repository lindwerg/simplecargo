## 2. Как это входит в расчёт тарифа (plain-Russian)

- **Что это:** K1 — классовый коэффициент. Это **безразмерный множитель** (не рубли, не копейки).
- **К чему применяется:** только к **И-части** тарифа — инфраструктурно-локомотивная составляющая (плата за пользование инфраструктурой РЖД и локомотивную тягу). Не к В-части (вагонная составляющая) и не к НКО. Это прямо следует из заголовка таблицы: «…на перевозку грузов **по инфраструктуре РЖД**…».
- **Шаг расчёта:** применяется на этапе формирования И-части по тарифному расстоянию. Сначала определяется тарифный класс груза (Табл. N 1, по коду ЕТСНГ), затем расстояние перевозки в км, затем выбирается строка-пояс из Табл. N 2 → коэффициент. Базовая И-ставка по поясу умножается на K1.
- **Единица:** множитель к рублям И-части.
- **Зависимость от расстояния:**
  - **Класс 1** — единственный класс, где K1 **зависит от расстояния**: убывает с 0,75 (до 1200 км) до 0,55 (5001 км и более). Чем дальше, тем дешевле инфраструктура за тонно-км (дегрессия).
  - **Класс 2** — всегда **1,00**, расстояние не влияет (нейтральный класс).
  - **Класс 3** — **не зависит от расстояния**: либо 1,74 (поименованный перечень ЕТСНГ, дорогие/высокодоходные грузы), либо 1,54 (остальные грузы третьего класса).
- **Выбор пояса для класса 1:** диапазоны **включительные по обеим границам** (например, ровно 1200 км → пояс «1–1200» → 0,75; 1201 км → следующий пояс → 0,74). Последний пояс «5001 и более» — открытый сверху.

## 3. Сверка с движком / сидами (что подтверждается, что расширяет/противоречит)

**Полностью совпадает с сидом** `scripts/seed-data/tr1-k1-full.json`:
- Все 21 пояс класса 1 (0,75 → 0,55) совпадают дословно с массивом `classCoeff`.
- Класс 2 = 1,00 «независимо от расстояния» совпадает.
- Класс 3: 1,74 (поименованный список) / 1,54 (остальные) совпадает; перечень ЕТСНГ в `etsngNote` сида **идентичен** дословному перечню источника.
- В сиде верхняя граница последнего пояса записана как `distToKm: 999999` — это инженерная кодировка дословного «5001 и более». НЕ противоречие, а представление открытого диапазона.

**Oracle-проверки (из `_meta.oracleChecks` сида, подтверждены формулой):**
- 699 км → класс 1 → K1 = 0,75 (пояс 1–1200). ✔
- 2444 км → класс 1 → K1 = 0,68 (пояс 2401–2600). ✔ (2444 oracle EXACT)
- 3108 км → класс 1 → K1 = 0,65 (пояс 3001–3200). ✔ (R-Тариф 3108 oracle EXACT, per-wagon 82816)

**Расширения / зоны риска (НЕ противоречия, но требуют внимания движка):**
1. **Применять K1 ТОЛЬКО к И-части.** Заголовок таблицы прямо ограничивает действие коэффициента инфраструктурой РЖД. Если движок ошибочно умножает на K1 весь тариф (И+В) или В-часть отдельно — это завышение для класса 3 (1,74) и занижение для класса 1 на дальних плечах. Проверить точку применения в коде.
2. **Класс 3 — бинарный выбор 1,74 vs 1,54 по коду ЕТСНГ.** Сид хранит перечень дословно, но конкретное соответствие «данный груз → 1,74 или 1,54» движок должен резолвить по коду ЕТСНГ против этого перечня (диапазоны включительные). `_meta.gaps` сида это и отмечает. Для щебня/нерудных (класс 1) это неактуально, но для прочих грузов класса 3 нужна точная маппинг-функция.
3. **K1 ≠ K3/K4/K5.** Табл. N 2 — это именно классовый коэффициент по поясам. Не путать с товарными поправочными (Табл. N 4 = K3), отправочными (Табл. N 5 = K4) и направленческими (Табл. N 3). `tr1-class-coeff.json` объединяет K1 (Табл.2) + K4 (Табл.5) в один файл — при использовании следить, чтобы движок брал из правильной таблицы.
4. **Класс 2 = 1,00 — нейтрально, но обязательно присутствовать в таблице** (иначе движок может не найти строку и упасть/подставить дефолт). Сид строку класса 2 содержит.

**Противоречий с источником не обнаружено.** Сид `tr1-k1-full.json` — точная машинная копия Табл. N 2; полная машинная таблица живёт там, данный rulebook-чанк фиксирует дословный первоисточник и точку применения.
