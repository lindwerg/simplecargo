## Что эта таблица ЕСТЬ (plain Russian, как входит в расчёт)

Это **скидка (уменьшение, вычитаемая величина) в рублях** к тарифной плате за контрейлерную перевозку, а **не множитель**. Это её принципиальное отличие от большинства таблиц ТР-1 (K1/K3/K4/K5 и направленческие коэффициенты Табл.3 — все они безразмерные множители). Здесь — фиксированная сумма в рублях, которая **вычитается** из рассчитанного тарифа.

- **Единица измерения:** рублей на один контрейлер (`руб./контрейлер`), отдельно для гружёного и для порожнего рейса. Контрейлер = автотранспортная единица (прицеп / полуприцеп / съёмный кузов / автопоезд), перевозимая на вагоне.
- **Условие применения:** вагон загружен **полным комплектом** контрейлеров («полными комплектами на вагон»). Это режим контрейлерной перевозки из **п.24** Приложения N 1 (методология контрейлерных перевозок) — см. `00-index-prikaz-894-25.md`, строка про «п.24 контрейлерные».
- **Две градации по типу/длине транспортного средства:**
  1. прицеп или съёмный автомобильный кузов **до 7,8 м включительно** → вычет **9965 ₽** (гружёный) / **6976 ₽** (порожний);
  2. полуприцеп (любой размер), либо прицеп/съёмный кузов **свыше 7,8 м**, либо автопоезд → вычет **15632 ₽** (гружёный) / **10937 ₽** (порожний).
- **Гружёный vs порожний:** выбирается по состоянию перевозимой автотранспортной единицы. Порожний рейс контрейлера (возврат прицепа/полуприцепа до или после перевозки груза) даёт меньшую скидку.

**Место в последовательности расчёта (п.16 + п.24):** сначала по общей методологии п.16 / п.24 считается базовая тарифная плата за контрейлерную отправку (тарифные схемы для контрейлерных перевозок — это **Таблица N 11**, ещё не зафетчена, см. индекс). Затем из полученной платы **вычитается** соответствующая сумма из Таблицы N 13 — по типу автотранспортной единицы и по признаку гружёный/порожний, умноженная на количество контрейлеров в комплекте. Итог округляется по п.15.5 (до рублей / 0,1 ₽, half-up — см. `tr1-rounding-rules.json`).

Это структурный аналог **Таблицы N 12** (такое же «уменьшение тарифов» для контейнерных отправок полными комплектами на вагон) — обе таблицы реализуют скидку за полную загрузку вагона комплектом единиц.

## EXTENDS / CONTRADICTS текущего движка и сидов

**Статус: НОВАЯ область, в движке НЕ реализована. Контрейлерные перевозки сейчас вне расчётного контура SimpleCargo (бизнес — щебень/нерудные повагонно, см. MEMORY).**

Что есть на диске сегодня (cross-ref):

- `scripts/seed-data/etsng-classes.json` — **уже содержит** номенклатуру контрейлерных единиц как грузы ЕТСНГ (строки ~38421–38502): «Автопоезд порожний/гружёный, используемый в системе контрейлерных перевозок», «Полуприцеп …», «Прицеп …», «Кузов съёмный …». То есть классификатор грузов знает про контрейлеры, **но скидка Табл.13 к ним не привязана**.
- `scripts/seed-data/tr1-coefficients.json` и `tr1-special-rules.json` — содержат текстовые упоминания контрейлеров, но **не** числовые вычеты 9965/6976/15632/10937. Эти четыре значения **отсутствуют в сидах** — это новый, ранее не captured факт.
- `docs/planning/rulebook/00-index-prikaz-894-25.md` — индекс помечал Таблицу N 13 как «TO FETCH (low)» (строка 45). Этот файл закрывает тот пункт; индекс следует обновить на «FETCHED / VERBATIM-CONFIRMED».
- `docs/planning/rulebook/tablitsa-n-3.md` (Раздел 4, круглый лес) упоминает «контрейлеры» только в контексте охвата заградительного коэффициента 70 — это другая таблица и другой механизм (множитель), **не пересекается** с Табл.13.

**EXTENDS:** добавляет в систему понятие *аддитивной рублёвой скидки* (вычета), которого в текущей множительной модели движка нет. Если контрейлерные перевозки когда-либо войдут в продукт, потребуется:
1. сид-файл (предлагаемое имя `tr1-contrailer-discount-n13.json`) с 2 строками × {груж/порож};
2. ветка в движке, которая **вычитает**, а не умножает;
3. зависимость от ещё не зафетченной **Таблицы N 11** (тарифные схемы контрейлерных перевозок) — без неё базу для вычитания посчитать нельзя.

**CONTRADICTS:** прямых противоречий с существующими сидами нет (область не реализована). Единственная концептуальная коллизия — модель «всё есть множитель»: Табл.13 это нарушает (вычет в рублях), что нужно учесть при проектировании движка, чтобы скидку случайно не реализовали как коэффициент.
