## 3. Как это входит в расчёт тарифа (plain Russian)

Это **дополнительная фиксированная плата (сбор), а не множитель**. Она возникает только для перевозок, маршрут которых физически проходит через паромную переправу **Ванино – Холмск – Ванино** (материк ↔ остров Сахалин). Накатка/выкатка — это заезд/съезд гружёного вагона (или контрейлерной/контейнерной единицы) на/с парома.

Место в последовательности расчёта (п.16 ТР-1):

1. Сначала рассчитывается **основной провозной тариф** по формуле раздела II (тарифная схема × расстояние × классные/направленческие коэффициенты K1…K5 и т.д.) — без участия этой таблицы.
2. **Поверх** готового тарифа добавляется **плата по Таблице N 20** как отдельное слагаемое в рублях, если маршрут включает переправу Ванино–Холмск.
3. Единица начисления зависит от вида отправки:
   - повагонная/групповая/маршрутная и контрейлерная → **за вагон / за единицу** (76 392 руб.);
   - контейнерная → **за каждый контейнер** по его типу/массе (7 636 … 76 392 руб.);
   - мелкая отправка → **за каждую тонну** груза (7 636 руб./т).

То есть это плоский сбор, прибавляемый к итоговой провозной плате, без зависимости от расстояния и без процентного коэффициента. Округление итога — по общим правилам п.15.5 (до рублей / 0,1 руб., half-up), уже на стадии суммирования с основным тарифом.

## 4. Что это РАСШИРЯЕТ или ПРОТИВОРЕЧИТ в текущем движке / сидах

**Статус в индексе:** в `docs/planning/rulebook/00-index-prikaz-894-25.md` строка `| 20 | Плата за накатку/выкатку на паромной переправе | — | TO FETCH (low) |` — на диске нет seed-файла для этой платы. Этот документ закрывает фетч; сид-файл всё ещё отсутствует.

**Cross-ref сид-файлов (grep по `паром`, `накат`, `выкат`, `Ванино`, `Холмск`):**

- `scripts/seed-data/tr1-special-rules.json` — **нет** упоминаний переправы/накатки. → Сбор Таблицы N 20 в движке **не реализован**. Это РАСШИРЕНИЕ: новое плоское слагаемое, которого нет в текущей формуле.
- `scripts/seed-data/uzel-graph-gapfill.json:69` содержит участок `"ХОЛМСК-СЕВЕРНЫЙ ИЛЬИНСК-САХАЛИНСКИЙ"` — то есть сахалинская сеть в графе уже есть, но **признак «маршрут пересекает паромную переправу Ванино–Холмск»** в графе/движке отсутствует. Чтобы применить эту плату автоматически, понадобится флаг ребра «паромная переправа Ванино↔Холмск» (или явный список станций по обе стороны: Ванино — материк, Холмск/сахалинские станции — остров).
- Ни один из тарифных коэффициентных сидов (`tr1-k1…k5`, `tr1-class-coeff*`, `tr1-i-belts*`, `tr1-v-belts*`, `tr1-empty-run*`) к этой плате отношения не имеет — она **не множитель**, а отдельный рублёвый сбор; вписывать её в коэффициентные таблицы НЕЛЬЗЯ.

**Противоречий с уже сверенной формулой (`reference-quotes-rtariff.json`) нет:** эталонные котировки R-Тариф, по которым калибрована формула, относятся к континентальным маршрутам без сахалинской переправы, поэтому отсутствие этого сбора не искажало сверку. Риск возникает только если в КП попадёт маршрут на/с Сахалина — тогда движок недосчитает 7 636…76 392 руб. на единицу.

**Рекомендация по реализации (НЕ выполнять здесь, только зафиксировать):**
1. Завести сид-файл, напр. `scripts/seed-data/tr1-ferry-vanino-kholmsk.json`, с 9 строками из раздела 2 (значения только из этой таблицы, без выдумывания).
2. Добавить в граф признак переправы Ванино–Холмск (флаг ребра/узла), чтобы детектировать пересечение.
3. В движке — отдельный шаг «надбавки/сборы поверх провозного тарифа», куда эта плата кладётся как плоское слагаемое по виду отправки и единице.
