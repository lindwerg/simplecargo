## 5. КАК ЭТО ВХОДИТ В РАСЧЁТ ТАРИФА (plain-Russian)

Таблица N 1 — это **классификатор груза**. Она даёт два входных параметра тарифной формулы (Раздел II, п.16), которые движок берёт ДО любых ставок:

### 5.1 Тарифный класс груза (столбец 3) → выбор коэффициента K1 и K3
- Каждый груз отнесён к одному из **трёх тарифных классов** (1, 2 или 3) — это та самая дифференциация из п.10 Раздела I.
- Класс **не множитель сам по себе**. Он — **ключ выбора** в коэффициентных таблицах:
  - **Таблица N 2 (= K1)** — коэффициент по расстоянию: своя колонка для класса 1 / 2 / 3 (`scripts/seed-data/tr1-k1-full.json`).
  - **Таблица N 4 (= K3)** — поправочные коэффициенты для отдельных грузов (`tr1-k3-full.json`, `tr1-commodity-coef-verify.json`).
- Логика: класс 1 (сырьё: щебень, песок, уголь, руда) — самый дешёвый; класс 3 (готовая продукция: машины, химия, текстиль) — самый дорогой.
- **Где в движке:** на шаге выбора множителей расстояния/груза. Груз → код ЕТСНГ → класс → колонка K1 по поясу расстояния × K3 по грузу.

### 5.2 МВН — минимальная весовая норма (столбец 4) → расчётная (тарифная) масса
- МВН задаёт **нижнюю границу тарифной массы** одного вагона в тоннах. Тариф считается не от фактической, а от **расчётной массы**:
  > **billable-mass = max(фактическая загрузка вагона, МВН)** (см. `scripts/seed-data/tr1-min-weight-norms.json`, поле `_meta`).
- Формат МВН зависит от типа вагона (легенда из шапки столбца 4):
  - `г/п` — норма = **грузоподъёмность вагона** (т.е. грузи под полную вместимость; для тяжёлых сыпучих это фактический потолок);
  - число (напр. `40`, `53`, `58`) — фиксированная норма в тоннах;
  - комбинированная запись `кр, пв - г/п, пл - 46` — для крытого и полувагона норма = г/п, для платформы = 46 т.
- **Где в движке:** на шаге определения расчётной массы вагона, ДО умножения на ставку группы И (за рейс) и группы В (за пользование вагоном). Заниженная фактическая загрузка не снижает тариф ниже МВН.

### 5.3 Полная цепочка (для контекста)
```
груз → код ЕТСНГ (Табл.1) → { тарифный класс, МВН }
   ├─ МВН → расчётная масса = max(факт, МВН)
   └─ класс → K1(класс, пояс расстояния) [Табл.2] × K3(класс/груз) [Табл.4]
тариф = [ставка И (Прил.N2) + ставка В + порожний пробег (Табл.17)] × K1 × K3 × K4 ...
```
Таблица N 1 — это **самое начало** цепочки: без правильного класса и МВН все последующие ставки применяются не к тем числам.

## 6. РАСХОЖДЕНИЯ / РАСШИРЕНИЯ относительно текущего движка и сидов

Сверка вербатим-таблицы с `scripts/seed-data/{etsng-classes.json, tr1-classifier-full.json, tr1-min-weight-norms.json}`:

### 6.1 ПОДТВЕРЖДЕНО (сид совпадает с первоисточником)
- Сверены строки `011005, 013000, 231000, 232003, 281000, 254002, 252007, 234009` — **тарифный класс и МВН совпадают 1-в-1** с вербатим-таблицей.
- `275 из 355` кодов Таблицы N 1 присутствуют в `etsng-classes.json`. Оставшиеся «отсутствующие» — это НЕ пропуски: 7 из них — строки с оговоркой в ячейке кода (напр. `161005 (кроме ...)`), где сам код `161005` в сиде есть; прочие — групповые заголовки без класса.
- Щебень/нерудные (класс 1, грузоподъёмность в полувагоне) в движке настроены верно.

### 6.2 РАСШИРЕНИЕ (сид ШИРЕ первоисточника — это норма, не ошибка)
- **`etsng-classes.json` содержит 5036 записей** — это полный детализированный справочник ЕТСНГ до 6-значного кода конкретного груза.
- **Сама Таблица N 1 даёт только 280 тарифицируемых позиций** (укрупнённый уровень) + 75 групп.
- То есть seed — **достоверное надмножество**: класс/МВН в Табл.1 заданы на уровне позиции (напр. вся `161005` = класс 1), а 6-значные коды внутри позиции наследуют этот класс/МВН. Сид раскрывает это наследование явно. Расхождения значений НЕ обнаружено.

### 6.3 НА ЧТО ОБРАТИТЬ ВНИМАНИЕ (не противоречие, а edge-case для движка)
- **Оговорки «кроме кодов …»** в ячейке кода (напр. `161005 (кроме 161077, 161081, 161096)`): движок должен применять класс/МВН позиции ко всем подкодам, КРОМЕ явно исключённых — у них своя строка с собственным классом/МВН. Если классификатор движка наследует «вслепую», исключённые коды получат неверный класс. Проверить, что `etsng-classes.json` корректно переопределяет исключённые коды (на выборке `161077` сид даёт класс 1, МВН `кр - 63, пв - 63` — что совпадает с вербатим-строкой; механизм работает).
- **МВН-формат `г/п`** не является числом — его нельзя подставлять как тонну. Движок обязан резолвить `г/п` в грузоподъёмность КОНКРЕТНОГО вагона (она различается: полувагон ~69–75 т, платформа меньше). `tr1-min-weight-norms.json` должен хранить именно правило (`г/п` → грузоподъёмность модели вагона), а не зашитую константу.
- **Комбинированные МВН по типу вагона** (`кр, пв - г/п, пл - 46`): норма зависит от того, в какой вагон грузят. Для одного и того же груза 232003 в полувагоне МВН = г/п, на платформе = 46 т. Если движок берёт единое число вне зависимости от типа вагона — это потенциальная ошибка расчётной массы.
