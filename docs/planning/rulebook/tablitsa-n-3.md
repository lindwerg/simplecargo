## What this table IS (plain Russian, how it enters the calculation)

Таблица N 3 — это **направленческие (географические) поправочные коэффициенты**: множители, которые применяются к тарифной плате в зависимости от *того, КУДА/ОТКУДА и ЧЕРЕЗ ЧТО* идёт груз (Калининградская дорога; пограничные передаточные станции; конкретные участки Ледмозеро–Костомукша–Товарная и Махалино–Хасан). Это **не** товарная поправка на сам груз — товарная поправка живёт в **Таблице N 4 (K3)**.

Место в последовательности расчёта (п.16 Раздела II): после того как посчитана базовая ставка (группа И ± группа В) и применены расстоянческий K1 (Табл.2), товарный K3 (Табл.4) и отправочный K4 (Табл.5), направленческий коэффициент Табл.3 применяется как **дополнительный множитель к тарифной плате** для перевозок, попадающих под одно из перечисленных направлений. Единица — безразмерный множитель к рублёвой плате (как и прочие K). Для большинства внутрироссийских перевозок (в т.ч. щебень повагонно между обычными станциями) этот коэффициент **= 1 (не применяется)** — он включается только для калининградского транзита, погранпереходов и двух именованных лесных маршрутов.

## EXTENDS / CONTRADICTS vs текущий движок и seed-файлы

**EXTENDS (новый материал, на диске ОТСУТСТВУЕТ):**

- Таблица N 3 целиком отсутствует в seed-данных. Поиск по `scripts/seed-data/` (grep `калинингр|погранич|ледмозеро|хасан`) даёт совпадения **только** в географических данных (графы узлов, станционные CSV: `uzel-graph*.json`, `rzd-stations-20231230.csv`, `cis-stations-20201230.csv`, `kniga3-backbone*.json`, `special-distances.json`), а НЕ в тарифных коэффициентных файлах. То есть направленческие коэффициенты Табл.3 в движок пока не заложены.
- Это закрывает открытый вопрос из `docs/planning/rulebook/00-index-prikaz-894-25.md` (строка N=3: «PARTIAL — verify directional vs commodity split»): **Таблица N 3 (направленческие) полностью ОТДЕЛЕНА от товарного K3.**

**Подтверждённое разделение Табл.3 (направленческие) vs Табл.4 (товарный K3):**

- `scripts/seed-data/tr1-k3-full.json` — это **Таблица N 4** (`.../tablitsa-n-4/`), товарная поправка K3 на конкретный груз/группу ЕТСНГ (например щебень/нерудные 231–236 → K3 = 0,77 + п.1.5 ×0,909). Это НЕ направленческий коэффициент.
- `scripts/seed-data/tr1-class-k3-full-verify.json` содержит `directionQualifier`-поля (`0,895 — за исключением … сухопутных пограничных переходов …`; `экспортные направления … 1,05`). Эти оговорки относятся к **Таблице N 4 (товарный K3)** — это экспортные признаки внутри товарной поправки, а **не** ячейки Таблицы N 3. Их нельзя путать с направленческими коэффициентами Табл.3.
- Вывод: товарный коэффициент (что за груз) и направленческий коэффициент (куда/через что) — это два независимых множителя в п.16, и в перемножении они оба входят в тарифную плату. Текущий движок применяет товарный (Табл.4), но **не** применяет направленческий (Табл.3).

**CONTRADICTS:** прямых противоречий с захваченными verbatim-правилами не обнаружено. Расхождение здесь — это **пропуск** (Табл.3 не реализована), а не конфликт значений.

**Влияние на основной кейс SimpleCargo (щебень/нерудные, повагонно, обычные станции РФ):** НУЛЕВОЕ при типовой внутренней перевозке — Табл.3 для неё даёт множитель 1 (не применяется). Коэффициенты Табл.3 включаются только при: (а) калининградском транзите, (б) экспорте/импорте через сухопутные погранпередаточные станции, (в) лесных маршрутах Ледмозеро–Костомукша–Товарная / Махалино–Хасан. Если SimpleCargo начнёт считать экспортные направления через погранстыки, Раздел 3 станет обязательным — и тогда его числа надо взять из сырого HTML/R-Тариф, а не из этого захвата.
