## 2. Что это значит и КАК входит в расчёт тарифа (простыми словами)

«Отправка в составе поездного формирования, не принадлежащего перевозчику РЖД» (СПФ) — это особый вид отправки, при котором клиент/оператор предъявляет к перевозке **собственный (не РЖД) поездной состав вместе со своим локомотивом**, а РЖД оказывает только услугу инфраструктуры (пропуск поезда). Это НЕ обычная повагонная/маршрутная отправка вагонами, где тягу обеспечивает РЖД.

Раздел I (см. `docs/planning/TARIFF_RULES_EXACT.md`, цитата п. установления вида отправки) прямо перечисляет СПФ как один из видов отправки наравне с повагонной, групповой, маршрутной и т.д.:

> «…повагонная, групповая, маршрутная, контейнерная, мелкая, сборная повагонная **или отправка в составе поездного формирования, не принадлежащего перевозчику РЖД**…»

Таблица N 18 — это **диспетчер тарифных схем** для этого вида отправки: она НЕ содержит самих ставок (рублей/коэффициентов), а лишь указывает, ПО КАКОЙ тарифной схеме (по её номеру) считать каждый элемент поезда. Сами ставки этих схем (110–115) лежат в схемных таблицах И-составляющей / поездной части ТР-1, а не здесь.

Место в пайплайне расчёта (шаг «выбор схемы»):

1. Определён вид отправки = «в составе поездного формирования».
2. Определён вид тяги: **тепловозная** (дизель) или **электрическая**.
3. По Таблице N 18 каждому элементу состава присваивается номер схемы:
   - локомотив: тепловоз → **110**, электровоз → **113** (плата за пропуск локомотива по инфраструктуре);
   - вагон с грузом: тепловозная тяга → **111**, электрическая → **114**;
   - порожний вагон: тепловозная тяга → **112**, электрическая → **115**.
4. Тариф поезда = сумма по элементам: (схема локомотива) + Σ(схема груженых вагонов) + Σ(схема порожних вагонов), каждая схема — функция расстояния (и веса/осности по своим схемным таблицам), далее к результату применяются общие коэффициенты раздела (K-коэффициенты, индексация и т.д.).

Единица: номер схемы — это **ключ выбора** (безразмерный идентификатор), а не множитель. Множитель/ставку даёт уже выбранная схема по расстоянию.

Ключевое отличие от обычной отправки вагонами:
- В обычной отправке РЖД везёт своим локомотивом → тариф включает И-составляющую (схема N8 и т.п.) + вагонную составляющую (В-схемы / Таблица N 17 на порожний). Локомотив отдельной строкой НЕ тарифицируется — он «внутри» инфраструктурно-локомотивной ставки.
- В СПФ локомотив клиента — отдельный тарифицируемый элемент (схемы 110/113), а вагоны идут по «поездным» схемам 111/112/114/115, а не по N8/В-схемам.

## 3. Расширения и противоречия относительно текущего движка/seed

**Главное: это ГЭП (пробел), а не противоречие.** Схемы 110–115 и сам вид отправки «поездное формирование» в текущем seed/движке НЕ реализованы.

Сверено по диску:

- `scripts/seed-data/tr1-scheme-classifier.json` — 11 строк, измерения только `ownership ∈ {own, rzd}`, `shipment = wagon/...`, и схемы вида `N8 / N8(1) / N25 / N25(1) / N26` (И-часть) + `В1/В3` (вагонная составляющая). **Нет** ни поля «вид тяги» (тепловоз/электровоз), ни строки про отдельный локомотив, ни схем 110–115.
- `scripts/seed-data/tr1-scheme-classifier-extended.json` — расширенная привязка вагон→схема по разделу II (пп.16-23,27) и Таблицам N6/N7; «поездного формирования» не покрывает.
- Поиск `grep` по всем `scripts/seed-data/*` строк `"110".."115"` и `поездн|формировани` — НЕТ ни одного попадания со схемами 110–115 (единственные совпадения слова «поездн» — в несвязанном README по книге 3 СНГ).
- В `docs/planning/TARIFF_RULES_EXACT.md` вид отправки СПФ лишь УПОМЯНУТ в вербатим-цитате раздела I как один из возможных, но методика его расчёта не расписана.

**Вывод для движка:**
- Текущий калькулятор SimpleCargo считает обычные отправки вагонами (груз класса 1/2/3, своя/общая принадлежность, порожний пробег) — это сценарий, релевантный щебню/неруду и общему парку. СПФ (свой поезд + свой локомотив) — это сценарий крупного оператора с собственной локомотивной тягой, который для текущей бизнес-модели, скорее всего, вне периметра.
- Если СПФ когда-нибудь потребуется, нужно: (а) добавить измерение «вид отправки = train_formation» и «вид тяги = diesel/electric»; (б) завести seed-таблицы ставок схем 110–115 из соответствующих схемных страниц ТР-1 (их в Таблице N 18 НЕТ — она только маппинг); (в) реализовать суммирование по элементам поезда (локомотив + Σгруж. + Σпорож.).

**Противоречий с уже зашитыми схемами (N8, N25, В1/В3) НЕТ:** схемы 110–115 — отдельное непересекающееся пространство номеров для другого вида отправки. Никакую существующую ячейку belt/коэффициента эта таблица не меняет и не отменяет.

**Не выдумано:** в Таблице N 18 нет числовых ставок и коэффициентов — только 6 номеров схем (110–115) и привязка к виду тяги/элементу. Никакие рубли/коэффициенты по этой странице в seed не вносятся.
