## 1. Что это за таблица (роль в расчёте)

Таблица N 16 — это **классификатор тарифной схемы (scheme classifier)**. Она НЕ содержит
ни рублёвых ставок, ни коэффициентов. Её единственная функция в движке: по двум входным
признакам отправки

1. **тип/осность вагона или транспортёра** (строка таблицы), и
2. **степень и вид негабаритности груза** (колонка таблицы),

— вернуть **номер тарифной схемы** (целое число 34…83). Этот номер затем подаётся в Приложение N 2,
где по нему берётся базовая ставка (для транспортёров — «за один транспортёр», для платформ/полувагонов —
«за один вагон», см. заголовки разделов Прил. N 2).

**Место в пайплайне расчёта:** шаг «определение тарифной схемы» (до похода в пояса дальности
Приложения N 2). Единица результата схемы — не из этой таблицы, а из заголовка соответствующего
раздела Прил. N 2 (рубли за транспортёр / за вагон).

**Расшифровка соответствия «колонка → степень негабаритности» (КРИТИЧНО для движка):**

- Колонка негабарита **K1** = степени **1-2 нижняя**, **1-3 боковая**, **1-2 верхняя**
- Колонка негабарита **K2** = степени **3-4 нижняя**, **4 боковая**, **3 верхняя**
- Колонка негабарита **K3** = степени **5 нижняя**, **5 боковая** *(верхней нет)*
- Колонка негабарита **K4** = степени **6 нижняя**, **6 боковая** *(верхней нет)*
- Отдельная колонка слева = **Габаритные грузы** (без негабаритности)
- Отдельная колонка справа = **Сверхнегабаритные грузы**

(К1…К4 — это мои метки для удобства движка, в источнике колонки безымянные; верно лишь то, что
в одной физической колонке сгруппированы перечисленные степени нижней/боковой/верхней негабаритности.)

## 5. Как это входит в расчёт тарифа (plain-Russian)

1. **Вход:** род подвижного состава (4-осная платформа/полувагон ИЛИ транспортёр с N осями)
   + степень и вид негабаритности груза (по габаритам: нижняя/боковая/верхняя, степень 1…6,
   либо «габаритный», либо «сверхнегабаритный»).
2. **Поиск строки:** по типу/осности подвижного состава выбирается строка (одна из 10 строк-данных).
   Для транспортёров важно различать раздел 2 (в составе грузовых поездов) и раздел 3 (с отдельным
   локомотивом РЖД) — это разные строки с разными номерами схем при одинаковой осности (напр.
   16-осный: раздел 2 → схемы 54-58, раздел 3 → схемы 59-63).
3. **Поиск колонки:** по степени/виду негабаритности выбирается колонка (Габаритные / K1…K4 /
   Сверхнегабаритные). Степень → колонка по сопоставлению из раздела 3 этого файла.
4. **Результат:** на пересечении — **номер тарифной схемы** (34…83).
5. **Дальше (вне этой таблицы):** номер схемы → Приложение N 2 → базовая ставка по поясу дальности.
   Единица берётся из заголовка раздела Прил. N 2: для платформ/полувагонов «за один вагон»,
   для всех транспортёрных схем 39-83 — «за один транспортер».

Таблица N 16 — это **только маппинг (тип × степень) → № схемы**. Никаких рублей, осей-множителей,
весовых норм или коэффициентов здесь нет.

### 6.2 `tr1-scheme-classifier.json` / `tr1-scheme-classifier-extended.json` — ПРОВЕРИТЬ ПОКРЫТИЕ

Эти файлы — общий классификатор схем движка. Нужно убедиться, что в них заведены ветки:
негабарит на 4-осных платформах/полувагонах (34-38) и все транспортёрные ветки (39-83) с разбивкой

**Действие для движка:** свести классификатор (Таблица N 16) и belts (Прил. N 2) так, чтобы
каждая (осность × степень) → № схемы → существующий пояс ставок.

### 6.3 НЕ ПРОТИВОРЕЧИТ

Прямых противоречий между Таблицей N 16 и существующими сидами не обнаружено. Все номера схем
