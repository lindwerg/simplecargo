# Импорт якорных расстояний (анкеров) — руководство оператора

Якорь — это **проверенное деньгами или R-Тарифом** тарифное расстояние между двумя
станциями. Якоря живут в `scripts/seed-data/special-distances.json` (слой L1
финальной архитектуры — см. `docs/planning/DISTANCE_FINAL_ARCHITECTURE.md`) и
**перебивают любой расчёт движка**. Поэтому правило одно:

> **Импортируем только те километры, которые можем защитить квитанцией или
> расчётом R-Тариф. Уверенно-неверный км — худший исход.**

Эталонный пример якоря: Красный Сокол → Бологое-Московское = **801 км**,
выверен фактической оплатой до копейки (16 ваг × 34 906 ₽ × 1,22 НДС = 681 365,12 ₽).

## Откуда брать километры

1. **Квитанции / накладные (лучший источник).** В квитанции о приёме груза и в
   счёте РЖД указано тарифное расстояние. Если по нему реально заплатили — это
   зелёный якорь.
2. **R-Тариф.** Посчитайте пару станций в R-Тариф (актуальная версия, режим
   «кратчайшее расстояние»), возьмите км из результата. В поле «note» так и
   пишите: `R-Тариф 2026-06, скрин в ...`.
3. **НЕ брать:** km с агрегаторов/форумов, «по памяти», из открытой выгрузки
   Книги 3 2023 года (она ДО Приказа №313 и не учитывает обход скоростных
   линий — для этого есть отдельный жёлтый слой L3, не якоря).

## Формат файла

TSV (рекомендуется — имена станций содержат запятые) или CSV (`;` или `,`),
по строке на пару:

```text
origin	dest	km	note
021609	612709	2444	квитанция №... от 2026-05-30
Лихославль	Окуловка	123	R-Тариф 2026-06, скрин в Drive
```

- **origin / dest** — 6-значный код ЕСР **или** точное название станции
  (как в справочнике `rzd-stations-20231230.csv`; буква «ё» = «е», регистр не
  важен). Если название неоднозначно (есть на нескольких дорогах) — скрипт
  откажет и попросит код ЕСР.
- **km** — целое положительное тарифное расстояние.
- **note** — обязателен по смыслу: чем подтверждено (№ квитанции, скрин R-Тариф).
- Строка заголовка определяется и пропускается автоматически.
- Экспорт из Excel: «Сохранить как → Текст (разделители — табуляция)».

## Запуск

```bash
# сначала посмотреть, что произойдёт (файл не трогается):
node scripts/import-anchors.mjs anchors.tsv --dry-run

# применить:
node scripts/import-anchors.mjs anchors.tsv
```

Скрипт печатает сводку: `added` / `updated` / `unchanged` / `refused`.

## Правила поведения

- **Всё или ничего.** Любая ошибка валидации (неизвестный код, кривой км,
  дубль пары в файле) — импорт целиком отменяется, файл не меняется.
- **Идемпотентность.** Повторный импорт того же файла ничего не меняет
  (`unchanged`). Направление пары не важно: `A→B` и `B→A` — одна пара.
- **Обновление.** Если пара уже есть с другим км — км/заметка/источник
  обновляются с громким предупреждением `старый → новый`.
- **Защита выверенных якорей.** Строки, чей `source` не начинается с
  `operator-import` (например, Бологое 801 — выверен оплатой), скрипт
  **отказывается** перезаписывать. Перебить можно только флагом `--force`
  и только если новый км подтверждён квитанцией.
- Каждая импортированная строка получает `source: "operator-import-<дата>"`.

## После импорта

Перезапустите приложение (или дождитесь редеплоя): граф расстояний кэшируется
при первом запросе, новые якоря подхватятся после рестарта. Проверка:

```bash
npx vitest run src/lib/distance
```

Контрольные оракулы (2444 / 699 / 3108 / 1432 / 801 и тарифные) должны
остаться зелёными — импорт добавляет пары, но не должен ломать эталоны.
